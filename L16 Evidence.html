Modularity Theory: Domain Specificity Fodor (1983); Applied by Carey, Spelke Theoretical framework Modules process only specific content (objects, not language; faces, not tools). Prediction: Object knowledge shouldn't transfer to other domains. Abilities within domain should co-emerge (unity, solidity, individuation all at 4mo). Evidence: Object-related abilities (unity via common motion, solidity understanding, object individuation via spatiotemporal cues) all emerge ~4 months. Carey's argument: Co-emergence supports unified module activating. Constructivism has no principled reason for simultaneity (each learned from different experiences). Unity, solidity, individuation: all ~4mo. Spatiotemporal continuity: 9mo (outlier - nativists explain as separate development). NATIVIST Systematicity argument: If multiple conceptually-related abilities emerge simultaneously, suggests single underlying system (module) coming online. Constructivists counter: Could be coincidence, shared prerequisites, or hierarchical construction reaching threshold together. "When three abilities unlock at once, it's one key turning three locks â€” module, not learning."

Constructivists counter: Motion is most reliable cue in environment (learned first); other cues added later via experience. "Module has a keyhole (common motion) â€” only the right key turns it on."
Modularity Theory: Core Knowledge Application Spelke & Kinzler (2007); Carey (2009) Theoretical framework Once triggered, module applies innate principles: (1) Persistence (objects exist continuously), (2) Solidity (objects don't interpenetrate), (3) Cohesion (object parts move together), (4) Contact (objects act on each other via contact). NATIVIST Core knowledge: Principles are unlearned (too abstract, too early, too little experience to derive from statistics). Constructivists counter: Massive prenatal + early postnatal experience; statistical learning rapid; principles could emerge from lower-level regularities. "Core knowledge: persistence, solidity, cohesion, contact â€” four rules the module knows from birth."
